     * scheme_candidates = [] 강제

2. JSON schema normalizer 추가 (p55, p57 list 버그 해결)
   - normalize_response() 함수가 Gemini 응답을 항상 dict로 정규화
   - list 응답 → 첫 원소 자동 unwrap
   - 필수 key 누락 → 기본값 자동 보충
   - page_kind 유효값 검증
   - scheme_candidates가 list가 아닌 경우 [] 로 보정

3. SMILES 정책 강화
   - smiles_confidence < 0.85 → reactant_smiles=None, product_smiles=None 강제
   - meta/toc 페이지: 정규화 단계에서 SMILES 전면 NULL
   - pipeline 코드에서도 None, None 하드코딩 유지

4. 빈 페이지 placeholder 제거
   - scheme_candidates=[] 이면 DB에도 0행 (fake scheme#1 생성 금지)
   - raw JSON / DB / manifest 일관성 유지

5. manifest fieldnames 보강
   - error 칼럼 포함 (MF_FIELDS 정의)
   - extrasaction="ignore" 추가로 CSV 생성 버그 방지
   - page_kind, scheme_count, extract_count 칼럼 추가

6. page_images에 page_kind 칼럼 저장
   - DB에서 page_kind 기반 필터링 가능
   - 추후 promote 필터 기준으로 활용

7. 프롬프트 v2 (prompt_named_reactions_v2.txt)
   - STEP 1: page_kind 분류 지시 추가
   - toc/meta_explanatory: scheme_candidates=[] 명시
